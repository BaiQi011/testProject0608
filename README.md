# 前端案例题说明文档

## 1. 使用了哪些 AI 工具？在哪些模块中？

本项目开发过程中主要使用了 AI 辅助工具 Codex（GPT-5.4）进行协作开发，使用方式集中在以下几个模块：

- 项目初始化与工程搭建：辅助生成 Vue 3 + TypeScript + Pinia + Vue Router + Less + Ant Design Vue + ECharts 的基础工程结构。
- 登录页开发：辅助生成登录表单、页面布局、按钮与提示区域的初始实现。
- 工单工作台开发：辅助完成工单表格、角色权限控制、管理员删除功能以及页面整体结构。
- 图表模块开发：辅助生成 ECharts 柱状图配置，并实现工单删除后图表自动联动刷新。
- 样式优化：辅助调整登录页居中布局、说明模块样式，以及表格状态标签和图表标题位置等界面细节。

AI 在本项目中主要承担“生成初稿 + 协助迭代调整”的作用，核心业务逻辑、交互取舍和界面细节是结合题目要求逐步修改完成的。

## 2. 哪些代码由 AI 生成？你做了哪些修改？

AI 辅助生成的代码主要包括以下几部分：

- Vite 工程基础配置文件，例如 `package.json`、`vite.config.ts`、`tsconfig.json`。
- Vue 页面基础结构，例如登录页、工作台页、图表组件、Pinia store、路由配置。
- ECharts 的柱状图基础配置，以及 Ant Design Vue 表格、表单的初始写法。

在 AI 生成初稿后，我结合实际效果和题目要求做了多轮修改，主要包括：

- 调整登录页结构：将原本左右分栏的登录页改成单卡片居中布局，并将说明信息移到登录区下方。
- 优化说明模块样式：让辅助说明区域在视觉上更统一，并保持横向居中。
- 微调图表表现：上移图表标题，避免与 y 轴的 `Hours` 文本重叠。
- 调整表格状态样式：交换 `Yes` 和 `No` 的标签颜色，使展示效果更符合当前预期。
- 完善权限逻辑：区分管理员与普通用户，确保删除按钮仅管理员可见，并在删除后同步更新图表数据。
- 构建与类型修正：修复表格列类型定义问题，保证项目能够通过 `npm run build`。

因此，这个项目并不是直接复制 AI 输出，而是在 AI 生成基础上，结合页面效果和功能要求不断修正、重构与优化后的结果。

## 3. 哪一部分你认为最难或最满意？

我认为本项目中最满意的部分是“工单表格与图表联动”这一块。

原因在于这个模块虽然功能不复杂，但比较能体现前端页面开发中的完整思路：

- 既要展示表格数据，又要根据相同数据源生成统计图表；
- 删除工单后，不只是更新表格，还要同步更新图表统计结果；
- 同时还要叠加权限控制，确保普通用户和管理员看到的操作入口不同。

这部分最终通过 Pinia 统一管理工单数据，并使用计算属性生成项目工时统计，使表格和图表都依赖同一份状态，因此删除记录后可以自然联动更新。这个实现方式结构比较清晰，后续如果扩展筛选、编辑或更多图表，也会比较容易维护。

相比之下，最需要反复调整的是登录页和说明模块的视觉布局，因为它涉及页面重心、信息层级和展示简洁度，需要根据实际效果不断微调，而不是只看代码逻辑是否正确。
